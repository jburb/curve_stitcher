Mashrabiya Debug Next Steps
===========================

Purpose
-------
Use Mashrabiya face debug labels and the exported face snapshot to define or refine shape picking and fill classification logic (star, petal, point) in a deterministic way.

Primary Window Helper
---------------------
window.exportMashrabiyaFaceSnapshot()

This helper is exposed on window and returns a JSON-friendly snapshot of face diagnostics for the current Mashrabiya geometry.

Suggested Workflow
------------------
1) Open Mashrabiya and set the target geometry context
   - Switch to the Mashrabiya experience.
   - Set the fold and geometry mode you are targeting before collecting diagnostics.

2) Ensure debug labels are visible
   - Keep mashrabiyaDebugLabelsEnabled = true while tuning.
   - On-canvas diagnostics show labels like: F{id} S/P/T b{bin}.

3) Capture diagnostic snapshot
   - In DevTools console, run:
     window.exportMashrabiyaFaceSnapshot()
   - IMPORTANT: As of 20260629, this function call when run as is returned a null view.center, so the following workaround was used to call it successfully:
      paper.setup(document.getElementById('myCanvas'));
      ({
         projectsCount: paper.projects.length,
         hasProject: !!paper.project,
         hasView: !!paper.view
      });
      window.exportMashrabiyaFaceSnapshot();

   - The snapshot includes, per face:
     id, bin, classification, radius, angle, hasScaffold, hasStarThread,
     hasCircleEdge, starEdgeCount, areaAbs, centroid, vertexCount.

4) Correlate labels with snapshot
   - Compare canvas labels against exported face entries.
   - Identify exactly which faces are misclassified and which signals distinguish them.
   - Focus on bin + radius + scaffold/thread flags first.

5) Tune picker stages in geometry builder
   - Edit the face-picking sections in buildMashrabiyaRosetteGeometry:
     a) Star face selection
     b) Petal face selection
     c) Triangle/point face selection
   - Adjust candidate filters and expandComponent constraints:
     - steps
     - maxAngleDrift
     - radius/scaffold/star-thread predicates

6) Recompute and verify
   - Re-run window.exportMashrabiyaFaceSnapshot() after each logic change.
   - Verify summary counts and per-face classifications move in the intended direction.
   - Confirm visual fill output still maps correctly to starRegions/petals/pointRegions.

Practical Heuristic
-------------------
Prefer deterministic thresholds by bin + radius + scaffold/thread flags over ad hoc visual tweaks.
Use debug labels to spot errors quickly, and the snapshot to encode stable logic.

Reference Points in stitchlab.html
----------------------------------
- buildMashrabiyaRosetteGeometry
- drawMashrabiyaFaceDiagnostics
- buildMashrabiyaFaceSnapshot

- window.exportMashrabiyaFaceSnapshot assignment

Notes
-----
- Keep debug labels enabled during tuning, then disable for release.
- Re-test across multiple folds to avoid overfitting rules to one configuration.
